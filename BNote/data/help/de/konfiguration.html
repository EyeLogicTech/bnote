Ab BNote Version 2.4 kann das System dynamisch konfiguriert werden. Im folgenden sind die Einstellungsmöglichkeiten genauer erläutert. Es empfielt sich nur versierte Benutzer Zugriff zu diesem Modul zu gestatten.

Parameter

Einzelne Einstellungen (Parameter) werden direkt beim Start des Moduls angezeigt. Durch einen Klick auf den Parameter wird ein Formular zur Veränderung des Parameters angezeigt. Durch einen Klick auf "OK" wird der neue Wert des Parameters gesetzt. Über "Zurück" gelangt man wieder zur Einsteigsansicht ohne zu speichern.

Parameter Beschreibung
Probenbeginn Legt die Uhrzeit fest an denen Proben in der Regel beginnen. Diese Uhrzeit ist beispielsweise die Voreinstellung für neue Proben.
Probendauer in min Legt die normale Dauer einer Probe in Minuten fest.
Standardgruppe Jeder neu registrierte Benutzer wird dieser Gruppe zugeordnet.
Automatische Benutzeraktivierung Ist die automatische Benutzeraktivierung eingeschaltet kann jeder der Zugriff zur Adresse dieses Systems hat sich mit Hilfe einer E-Mail-Adresse anmelden und auf Daten zugreifen. Die Aktivierung ist nur am Anfang der Benutzung, solange sich Mitglieder neu registrieren, und mit eingeschränktem Zugriff aus dem Internet sinnvoll. Die Aktivierung sollte nach ca. einer Woche wieder deaktiviert werden. Ist die automatische Benutzeraktivierung inaktiv so muss jeder neue Benutzer im User-Modul freigeschaltet werden.
Benutzer Registrierung Wird dieser Wert auf "nein" geändert, so können sich neue Benutzer nicht selbst auf dem System registrieren, d.h. im öffentlichen Bereich von BNote ist die "Registrierung"-Option deaktiviert und ein direkter Linkaufruf führt zu einer Zugriffsverweigerung.
Share-Lesemodus für Nicht-Administratoren Um allen Benutzern außer Administratoren (also Benutzer/Kontakte die nicht in der Gruppe Administratoren sind) den schreibenden Zugriff auf Verzeichnisse und Dateien zu verweigern kann dieser Wert auf "ja" gesetzt werden.
Probenlänge angezeigen In der Regel wird die Probendauer in Minuten angegeben (Standard: Wert = ja). Wird der Wert auf "nein" gesetzt, so wird die Probendauer nicht in Minuten angegeben, sondern es wird der Endzeitpunkt festgelegt. Zusätzlich wird auf der Startseite zum Probenbeginn noch das Ende der Probe angezeigt.
Vielleicht-Teilname zugelassen Die Mitglieder können bei Proben und Auftritten ihre Teilnahme angeben (ja/nein/vielleicht). Soll die Option "vielleicht" nicht zur Verfügung stehen, so kann dieser Wert auf "nein" geändert werden. Standardmäßig ist die Vielleicht-Option zugelassen (Wert = ja).
Zip-Download für Ordner zulassen Ist auf dem Server die PHP-Erweiterung "zip" installiert, so kann dieses Feature aktiviert werden. Damit ist es möglich in Share-Modul Unterordner (keine Hauptordner, wegen Last) als Zip-Archiv herunterzuladen. Standardmößig ist das Feature aktiviert.
Anzahl Auftritte auf Startseite Werden viele Auftritte hinzugefügt, so wird der Benutzer mit Ankündigungen überhäuft. Um dies zu reduzieren, kann ein maximaler Wert festgelegt werden, wie viele Auftritte auf der Startseite angezeigt werden sollen. Sind mehr als diese vorhanden, wird ein Link "Alle anzeigen" mit entsprechendem Verhalten und eine Nachricht dazu eingeblendet. Das Feature kann mit dem Wert 0 deaktiviert werden. Nach der Installation steht der Wert auf 5.
Anzahl Proben auf Startseite Werden beispielsweise in einer Probenstrecke viele Proben hinzugefügt, so wird der Benutzer mit Ankündigungen überhäuft. Um dies zu reduzieren, kann ein maximaler Wert festgelegt werden, wie viele Proben angezeigt werden sollen. Sind mehr als diese vorhanden, so wird ein Link "Alle anzeigen" mit entsprechendem Verhalten und eine Nachricht dazu eingeblendet. Das Feature kann mit dem Wert 0 deaktiviert werden. Nach der Installation steht der Wert auf 5.
Anzahl Updates auf Startseite Um die Zahl der Updates, die für den Benutzer relevant sind zu reduzieren, kann ein maximaler Wert festgelegt werden, wie viele Updates auf der Startseite angezeigt werden sollen. Nach der Installation steht der Wert auf 5.
Diskussionen erlauben Zu Proben, Auftritten und Abstimmungen können Diskussionen erstellt werden. Dies ist insbesondere in kleineren Ensembles notwendig, da hier eine enge Abstimmung zwischen den Mitgliedern stattfindet. Für Schulen kann es jedoch notwendig sein, das Feature zu deaktivieren, da es zu rechtlichen Schwierigkeiten und (unnötigem) administrativem Aufwand kommen kann. Standardmäßig ist das Feature aktiviert.
Anzahl der Erinnerungen Gibt die Zahl der Erinnerungen zurück, die versandt werden vor Ablauf der Zusagefrist für Proben/Auftritte bzw. vor dem Abstimmungsende.
Benachrichtigungen aktiv Aktivierung der automatischen Erinnerungsfunktion. Der Server schickt über einen von BNote.info bereitgestellten Service Nachrichten an die Probenteilnehmer/Auftritteinladungen sowie Abstimmungsberechtigten sollten sie noch nicht abgestimmt bzw. ihre Teilnahme angegeben haben.
Erinnerungszyklus (jede X Tage) Anzahl der Tage die das System die Teilnehmer/Abstimmungsberechtigten benachrichtigt.
Google Maps API Key Der Schlüssel aus dem Reiter "Anmeldedaten" in der Google Cloud Console für die Google Maps APIs "Geocoding API", "Maps JavaScript API" und "Maps Static API". Bitte stelle sicher, dass die API aktiviert wurde. Google Developer Console
Notification Interface Key Ein zufällig generierter Schlüssel für dieses System.
Land Das Standardland für Adressen in BNote. Hier wird der 3-stellige ISO-3166 code benötigt, z.B. DEU oder AUT für Deutschland bzw. Österreich.
Sprache Angabe des Sprachschlüssels für das System, z.B. "de". Derzeit wird nur teilweise auch "en" für Englisch unterstützt.
Währung In dieser Währung wird die Organisation bilanziert. Hier kann auch ein Symbol oder ein vollständiger Währungsname stehen.
Standarddirigent Welche Person standardmäßig als Dirigent für neue Proben eingestellt wird.
Fehlgeschlagene Loginversuche protokollieren Wird dieser Parameter auf "ja" gesetzt (aktiviert), dann werden nicht erfolgreiche Anmeldeversuche in die Datei BNote/log/login_fail.log mit IP-Adresse protokolliert. Bitte stelle sicher, dass der Serverprozessbenutzer diese Datei schreiben darf.

Benutzerdefinierte Felder

Ab BNote 3.4 ist es möglich Kontakten, Proben, Auftritten, Stücken, Reservierungen, Termine und Orte weitere Felder hinzuzufügen. Um dies zu tun, gehe wie folgt vor:

Gehe zur Konfiguration, wähle "Benutzerdefinierte Felder". Eine Übersicht der Felder wird gezeigt.

Benutzerdefiniertes Feld hinzufügen

Klicke auf "Feld hinzufügen" um ein neues Feld einem der o.g. Objekte hinzuzufügen. Ein Formular erscheint:

Technischer Name Bitte verwende nur Buchstaben und Zahlen um dem Feld einen eindeutigen technischen Schlüssel zu geben, z.B. "INSTRUMENT2".
Name Singular/Plural Wie das Feld heißen soll, also z.B: "Zweites Instrument".
Freigegeben Ob ein Kontakt das Feld sehen darf (dann ja), sonst ist das Feld nur für interne Zwecke. Bitte beachte, dass Kontakte ihre freigegebenen Felder auch selbst bearbeiten dürfen.
Wertebereich Welchen Wert das Feld annehmen darf.
Objektbezug Welchem Objekt dieses Feld hinzugefügt werden soll.

Instrumente

Bei der Registrierung legt jedes Mitglied fest welches Instrument er/sie spielt. Da eine Bigband andere Instrumente als ein Orchester oder ein Chor hat, kann die Auswahl an Instrumenten konfiguriert werden. Diese Konfiguration öffnet man über die Schaltfläche "Instrumente" oben im Konfigurationsmodul.

Instrumente anzeigen

Auf der Startseite der Instrumentenkonfiguration werden alle Instrumente alphabetisch sortiert angezeigt. Die zur Nutzung konfigurierten Instrumente sind in auf dem Registrierungsformular sichtbar. Zu diesem gelangst du in dem du oben rechts auf "Logout" klickst, BNote erneut aufrufst und dann links im Menü neben dem Login auf "Registrierung" klickst.

Instrument hinzufügen

Auf der Konfigurationsseite der Instrumente ist oben eine Schaltfläche "Instrument hinzufügen". Ein Klick auf diese öffnet ein Formular mit Namen und Kategorie der Instrumente. Mit "OK" wird ein neues Instrument angelegt.

Instrumente bearbeiten

Wie in anderen Modulen auch können einzelne Einträge einer Tabelle mit einem Klick auf die entsprechende Zeile angewählt werden. Ein Klick auf ein Instrumente zeigt Infos zum Instrumente an und eine weitere Schaltfläche "Instrument bearbeiten". Das Formular entsprechend ausfüllen und mit "OK" die Änderungen bestätigen.

Instrumente löschen

Neben der Schaltfläche "Instrument bearbeiten" ist die Schaltfläche "Instrument löschen" in der Detailansicht eines Instruments. Nach einer Bestätigung dass man das Instrument wirklich löschen möchte, wird das Instrument entfernt. Wird ein Instrument von einem Kontakt verwendet, so kann das Instrument nicht entfernt werden!

Instrumentenfilter

Auf der Startseite der Instrumentenkonfiguration ist die Schaltfläche "Instrumentenfilter". Klickt man darauf öffnet sich eine Liste mit Instrumentengruppen die an- und abgewählt werden können. Nur die angewählten Instrumentegruppen werden im Registrierungsformular angezeigt.
